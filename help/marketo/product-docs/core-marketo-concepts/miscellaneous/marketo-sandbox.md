# Marketo沙箱 {#marketo-sandbox}

Marketo沙箱是生產環境中實作前，用於測試用途的額外例項。

>[!AVAILABILITY]
>
>並非所有客戶都購買過此功能。 如需詳細資訊，請連絡您的客戶成功經理。

如果Marketo沙箱已同步至生產執行個體，則無法將其同步至一般CRM。 使用CRM的沙箱進行同步，並遵循原始同步的所有相同步驟。

## 沙箱須知 {#things-to-know-about-sandboxes}

* 一旦您的Customer Success Manager設定沙箱並傳送邀請後，您必須使用與Marketo生產執行個體不同的電子郵件地址來登入。
* 如果您想要新增使用者，程式與在生產環境中新增使用者[的程式相同。 ](/help/marketo/product-docs/administration/users-and-roles/managing-marketo-users.md#create-users)同樣地，如果使用者已經擁有Marketo登入資訊，則必須使用不同的電子郵件地址。
* 您的Marketo沙箱一開始會是空的，但其功能與您的生產執行個體相同。
* 如果您在沙箱中建立方案，並想將其移至生產環境，則可執行[方案匯入](/help/marketo/product-docs/core-marketo-concepts/programs/working-with-programs/import-a-program.md)。
* 沙箱會受到限制，因此生產執行個體不會受到測試環境的不利影響。 每次行銷活動執行最多可傳送20封電子郵件。

>[!CAUTION]
>
>我們目前不支援Marketo Dynamics Sync的沙箱重新整理。 如果您需要重新整理Dynamics CRM沙箱，則需要新的Marketo沙箱。 如需詳細資訊，請連絡您的客戶成功經理。

## 執行個體復本 {#instance-copy}

您可以提交支援案例，要求一次性執行個體副本以填入您的沙箱。 但是，實例副本不會帶來&#x200B;_everything_。 如需詳細資訊，請參閱[Marketo支援](https://nation.marketo.com/t5/Support/ct-p/Support) 。

>[!NOTE]
>
>* 如果來源例項已與Microsoft Dynamics整合，則例項復本為&#x200B;**不支援**。
>* 如果您要變更原生CRM，則需要新的Marketo例項，且無法複製新Marketo例項的例項。 請改為與Marketo支援合作，探索匯入方案功能。
